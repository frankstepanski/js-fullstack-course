Software engineering has changed. AI tools are genuinely useful — they can help you understand errors, explore unfamiliar syntax, and move faster once you know what you're doing.

But here's what hasn't changed: **you still have to learn to think like a programmer.**

This course teaches you to be a strong, capable engineer first. That means learning to read, write, debug, and reason about code on your own — building the mental models that make AI tools actually useful instead of a crutch. AI is part of this course, but it's a tool you'll learn to use wisely, not something that does the learning for you.

> **A note on cognitive debt:** Leaning on AI before you understand the fundamentals creates a hidden cost — you produce code you can't explain, debug, or extend. That debt compounds. This course is designed to prevent it by ensuring you understand the *why* behind everything you build.

There is no shortcut here. Learning to code is slow and nonlinear. That's not a bug — it's how durable skills are built.

## 📘 Prerequisites

You only need:
- Basic computer literacy
- Curiosity and persistence
- Willingness to sit with confusion before reaching for answers

Software engineering is a skill learned through building, breaking, and improving code — not through reading about it.

## A Complete Journey Through the JavaScript Stack

### 🎨 Frontend Development
Build the parts of a website users see and interact with. This is where products are won or lost — a poorly built frontend drives users away no matter how good the backend is. You'll start with the fundamentals of HTML and CSS before moving into JavaScript and React, building up each layer with a real understanding of how it works.

- **HTML & CSS** — structure and style modern, accessible, responsive pages
- **JavaScript** — from fundamentals to advanced concepts (DOM, APIs, events, logic)
- **React.js** — build dynamic, component-based interfaces using hooks, props, and state

> 💡 React powers the UIs at Meta, Airbnb, and Netflix

### 🖌️ UX Design

Learn how to think like a designer before you write a line of code. Most developers skip this entirely — and it shows in the products they build. Understanding how users think, what they expect, and where they get confused makes you a better engineer and a more effective collaborator on any team.

- User-centered design principles and design thinking
- Wireframing and prototyping interfaces before building them
- Understanding user flows, accessibility, and visual hierarchy

> 💡 Design collaboration is now a standard part of the dev workflow — developers and designers working from the same source of truth.

### ⚙️ Intermediate & Advanced JavaScript

 This separates engineers who can build things from engineers who can build things and explain why they work. Async patterns, the event loop, and closures come up constantly in interviews and in real codebases — you'll understand all of them deeply, not just well enough to copy a Stack Overflow answer.

- **Asynchronous Programming**
  - Callbacks
  - Promises
  - `async` / `await`
  - How the **event loop** enables non-blocking execution
- **JavaScript Internals**
  - Call stack and memory model
  - Execution context and scope chain
  - Closures and persistent data
- **Developer Essentials**
  - ES Modules vs CommonJS
  - Error handling
  - Browser APIs & Fetch
  - Debugging strategies with breakpoints and stack traces

> 💡 Having a deep understanding of your programming language means you can debug confidently, optimize performance, and reason about code you didn't write

### 🔷 TypeScript Fundamentals

Learn how professional teams add **safety, clarity, and confidence** to JavaScript codebases. TypeScript has shifted from a niche preference to an industry standard — the majority of new JavaScript projects at mid-to-large companies are written in it. You'll learn to use it as a tool that catches mistakes early, not as a hurdle that slows you down.

- Why TypeScript exists and what problems it solves
- Primitive types, objects, arrays, and functions
- Type inference vs explicit typing
- Interfaces and type aliases
- Optional properties and union types
- Using TypeScript with React components
- Understanding compile-time errors vs runtime behavior

> 💡 TypeScript is now the default at most serious engineering teams

### 🧪 Testing Fundamentals

Learn how engineers verify that code works — and keeps working as applications grow. Testing is what gives you the confidence to change code without fear — knowing that if something breaks, you'll catch it before your users do. It's a habit that takes time to build, but once you have it, you'll wonder how you ever shipped code without it.

- Why testing exists and what problems it solves
- Types of testing: unit, integration, component, and end-to-end (E2E)
- Writing basic tests for JavaScript and React components
- Understanding test assertions and expectations
- How testing improves refactoring confidence and code quality

> 💡 Writing tests is what lets teams move fast without breaking things

### 🖥️ Backend Development

Learn how to build the systems that power applications behind the scenes. While the frontend is what users see, the backend is what makes it all actually work — handling data, enforcing rules, managing authentication, and keeping everything secure. Understanding both sides of the stack makes you a significantly more capable and hireable engineer.

- **Node.js** — execute JavaScript outside the browser
- **Express.js** — build fast, maintainable APIs
- **Databases**
  - **SQL (PostgreSQL/MySQL)** — relational schemas, joins, constraints
  - **MongoDB** — flexible document storage
- **Core Backend Concepts**
  - RESTful API design
  - Authentication & authorization
  - Environment variables & secure configuration

> 💡 Where business logic, security, and data all meet

### 🔧 Version Control with Git

Git is a foundational skill for every professional developer. It's how teams track changes, collaborate without stepping on each other, and maintain a clear history of every decision made in a codebase. You'll use Git from day one of this course — and every day of your career after it.

- Tracking changes and writing meaningful commits
- Branching, merging, and resolving conflicts
- Collaborating on shared codebases
- Using GitHub for remote repositories and pull requests

> 💡 Collaborating on code is a requirement, both individually and as part of a team

### 🐳 Docker

Learn how modern teams ship software consistently across environments. One of the most common friction points in real development is code that works locally but breaks in production — Docker eliminates that problem by packaging your app and its dependencies into a portable container. It's also your first window into how modern cloud infrastructure actually operates.

- Containers vs virtual machines — understanding the difference
- Writing Dockerfiles and building images
- Running applications in isolated, reproducible environments
- Why Docker matters for deployment and team collaboration

> 💡 Packaging your app and everything it needs into a container that runs the same everywhere

### 📦 Data Structures & Algorithms

Build the problem-solving foundation that powers technical interviews and efficient code. DSA teaches you to think about problems before you start solving them — evaluating trade-offs, choosing the right tool for the job, and writing code that holds up at scale. It's also the area most likely to come up in a technical interview, so arriving prepared here gives you a real edge.

- Core data structures: arrays, linked lists, stacks, queues, trees, and hash maps
- Common algorithms: sorting, searching, and recursion
- Big O notation and thinking about performance
- Practical application in JavaScript

> 💡 Developing the habit of thinking about trade-offs before writing code

## 🤖 Understanding AI: What Every Developer Should Know

AI tools are part of the modern development landscape, and you should understand them — both how they work and how to use them without undermining your own growth.

This section is taught with a specific intention: **literacy over dependency.**

### 🧠 How AI Actually Works

Before using AI tools, you'll learn what's happening under the hood — no math degree required.

- What AI, machine learning, and deep learning really mean (and how they differ)
- What **large language models (LLMs)** are and how they generate text
- Key concepts: **tokens**, **context windows**, **temperature**, and **embeddings**
- The difference between **training**, **fine-tuning**, and **prompting**
- Strengths and real limitations of modern AI — including hallucinations, bias, and knowledge cutoffs
- Ethical considerations: privacy, attribution, and responsible AI use

### 🛠️ Using AI as a Learning Tool — Without Letting It Learn for You

AI tools like GitHub Copilot, ChatGPT, Claude, and Cursor are genuinely useful — when used at the right moment and in the right way. You'll develop judgment for when to reach for them and when to sit with a problem first.

**Useful AI habits:**
- Using AI to *explain* something after you've already tried to understand it yourself
- Asking AI to review code you've already written and identify potential issues
- Using AI to explore unfamiliar APIs or summarize documentation
- Treating AI-generated code as a *starting point to read critically*, not a final answer

**What creates cognitive debt:**
- Asking AI to write code before you've thought through the problem
- Copy-pasting AI output without understanding what it does
- Using AI to skip the debugging process — which is where real learning happens
- Letting AI make architectural decisions you haven't reasoned through yourself

> 💡 The engineers who benefit most from AI tools are those who already have strong fundamentals. Your goal is to build those fundamentals first — then AI becomes a genuine multiplier.

### 🔌 Building One AI-Powered Application

In the final project of this course, you'll integrate a real LLM API into a full-stack application you've built from scratch. By that point, you'll have enough context to do it meaningfully — understanding what the API is doing, how the data flows, and how to handle it responsibly.

Topics covered in this module include:
- Calling LLM APIs (**OpenAI**, **Anthropic Claude**, and others) from a Node.js backend
- Securely managing API keys with `.env` files — and why you never expose them on the frontend
- Designing **system prompts** to shape model behavior
- Streaming responses to the UI for a real-time experience
- Understanding cost, latency, and rate-limit tradeoffs in production
- Introduction to **Retrieval-Augmented Generation (RAG)** — giving AI access to your own data
- Using AI **tool calling / function calling** to connect models to real app logic
