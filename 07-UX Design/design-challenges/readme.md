
## Design Challenges to Reinforce UX + CSS Skills

Now that you've explored **UX design principles** and practiced essential layout techniques like **Flexbox**, **CSS Grid**, and **responsive design**, it's time to put your knowledge into action with real-world layout challenges.

Each challenge mimics a realistic scenario where a developer would receive design assets from a UX/UI designer.

### What You'll Receive

For every challenge, you’ll be provided with:

- 📸 Sample images (product photos, icons, etc.)  
- 🎨 A basic **style guide** (fonts, colors, spacing rules)  
- 📝 Copy/text content

In real-world projects, a designer typically provides even more detailed artifacts to help developers build **pixel-perfect layouts**:

- ✅ **Wireframes or mockups** — Visual blueprints of the UI, often shared via tools like Figma, Sketch, or Adobe XD. These help you understand **layout, spacing, and content hierarchy** before writing any code.

- ✅ **Style guide** — A human-readable reference with rules for fonts, colors, spacing, and component styles (e.g., buttons, forms, navbars). Think of it as the visual language your code should follow.

- ✅ **Design tokens** — Machine-readable variables that store those same decisions (e.g., color `#0066ff`, spacing `16px`, font `Roboto 400`) in code-friendly formats (JSON, SCSS, etc.). These let developers apply styles **consistently and programmatically** across a codebase or design system.

- ✅ **Component documentation** — Describes reusable UI pieces, their states (default, hover, active, disabled), behavior, and interaction rules. Helps devs build components that match the designer’s intent.

- ✅ **Accessibility requirements** — Guidelines and specs to ensure your UI works for **all users**, including those using screen readers, keyboards, or assistive tech. This might include contrast rules, alt text requirements, or keyboard navigation.

- ✅ **User flow diagrams** — Maps of how users move from screen to screen (e.g., login → dashboard → settings). These help devs understand what paths users might take and what logic or routing needs to happen in the app.

These challenges give you a taste of how developers and designers work together in professional teams.

### Design Challenges

Each challenge is meant to help you master layout patterns, spacing, typography, and responsive behavior:

#### 1. [Simple Contact Us Form](./1-contact-form/)
- Inputs for name, email, and message + a submit button.
- Focus on accessible labels, spacing, and a clean layout.

#### 2. [Three-Column Grid Layout](./2-three-columns/)
- A flexible grid showing features, team members, or services.
- Practice equal-width columns and responsive behavior.

#### 3. [Order Summary Page](./3-order-summary/)
- A checkout-style layout with product summary, prices, and totals.
- Apply visual hierarchy and structured layout.

#### 4. [Basic Grid Layout](./4-basic-grid/)
- Display card in 3 separate areas using CSS Grid.
- Practice grid auto-fit/auto-fill and spacing techniques.

#### 5. [Signup Form](./5-signup-form/)
- Multiple fields (name, email, password) with validation UX.
- Practice keeping content and a form spaced apart.

#### 6. [Landing Page Hero](./6-landing-page/)
- A bold intro with headline, subtext, and hero image, and CTA button.
- Test layout centering, background images, and responsiveness.

#### 7. [Shopping Cart](./7-shopping-cart/)
- Display cart items, quantities, and totals.
- Align text, inputs, and sections cleanly.
